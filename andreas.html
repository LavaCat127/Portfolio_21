Problem Statement

Andrea’s Dream Rescue website is not designed to handle the increase in demand for kittens/cats to adopt. They need more foster parents, and an updated adoption page, as well as increase donations. The main goal was to update ​the website with a thoughtful design and graphic hierarchy so that users could engage with the organization’s needs.

02 Research Phase

For research phase, we planned and conducted 5 interviews, 25 surveys, and a stakeholder interview in San Francisco. Base on the data we’ve collected our users is most 25 - 34 years old women that are in a relationship and live in the bay area.

Heuristic Evaluation

Usability Testing

We did the Usability Testing Tasks to observe if users can easily navigate through the website. We came up with four tasks: Is the user able to choose a cat to adopt? Is the user able to access the application page? Is the user able to access the donation page and understand the donation process? Is the user able to successfully find the foster section?

Fortunately, the majority of the users finish tasks effortlessly. Apart from the tasks, we noticed the menu from the current website is not very accessible for users to browse on a desktop webpage. Moreover, some of the users complained about the adoption application form is too much to scroll through, and suggested it should have a progress bar and keep the questions minimum.

04 Ideation

Sketches

After consolidating all the information in fewer pages, we began our ideation process by sketching out ideas and discussing what we would focus on for each page, and then we discussed which ways to go.

Lo-Fi Wireframes

Lo-fi wireframes Usability Test

I performed another user testing to understand how easy it was for a user to find the right information when using the navigation. I found out that the “deep linking“ from the dropdown was not only better for the user to see all the available sections but also made it easier to access them. With the new design, the users were able to quickly find all the important content and access it with just a click or two.

Based on the user testing, the main things to be fixed were:

  • The Application is too long and has too many questions
  • Confusion around what the Donations help with

05 Final Design

Desktop

There’s a bigger emphasis on cat images and a consistent “warm, friendly, and dreamy” feeling across the whole website. To enhance the explanation of the adoption process we decided to use some simple illustrations to show what people should expect, it's more intuitive. Moreover, I divided the application form into 4 stages to make the filling process easier to follow, instead of seeing a very long and overwhelming form.

Mobile

For mobile devices, the approach was to use a responsive design. In addition, the sections were simplified to save space and avoid distractions.
